1. Qu'est-ce que n8n ? (et pourquoi pas Zapier)

n8n (prononcez "n-eight-n") est un outil d'automatisation de workflows créé en 2019 par Jan Oberhauser. Son principe : vous connectez des applications entre elles via une interface visuelle drag-and-drop, sans écrire une seule ligne de code — ou avec du code si vous en avez envie.

Concrètement : un nouveau lead remplit votre formulaire Typeform → n8n l'ajoute automatiquement dans votre CRM, envoie un email de bienvenue personnalisé, crée une tâche dans Notion et vous notifie sur Slack. Tout ça en 3 secondes, sans intervention humaine.

Là où n8n se distingue des alternatives, c'est sur trois points :

💡 Mon conseil terrain

Si vous débutez et que vous voulez juste connecter Gmail à Google Sheets, Zapier peut suffire. Mais si vous construisez une vraie infrastructure d'automatisation pour votre PME, commencez directement avec n8n. Vous éviterez la migration douloureuse plus tard.

2. Comment démarrer : Cloud vs Self-Hosted

Deux chemins s'offrent à vous pour démarrer avec n8n :

Option A : n8n Cloud (recommandée pour débuter)

n8n.cloud vous donne accès à une instance hébergée, sans installation. L'essai gratuit dure 14 jours, puis c'est environ 20 €/mois pour le plan de base (suffisant pour la plupart des PME).

  • Création de compte sur n8n.io en 2 minutes
  • Pas de serveur à gérer
  • Mises à jour automatiques
  • Support inclus

Option B : Self-Hosted sur Railway (gratuit)

Si vous voulez héberger n8n gratuitement, Railway est le chemin le plus rapide. Comptez 15 minutes de setup, zéro compétence DevOps requise :

  • Créez un compte sur railway.app
  • Déployez le template n8n officiel en 1 clic
  • Configurez les variables d'environnement (N8N_ENCRYPTION_KEY, identifiants)
  • C'est en ligne et gratuit jusqu'à 5 $/mois de consommation
⚠️ Attention self-hosted

En self-hosted, vous êtes responsable des sauvegardes et des mises à jour. Pour une utilisation critique en production, pensez à activer les backups automatiques ou passez en cloud managé.

Le Trigger — Le déclencheur

Tout workflow commence par un trigger : un événement qui lance la machine. Ça peut être un événement temps réel (nouveau formulaire soumis, email reçu, webhook déclenché) ou un cron planifié (chaque lundi à 9h, toutes les 30 minutes, etc.).

Le Node — L'action unitaire

Chaque étape de votre workflow est un node. Un node peut lire ou écrire dans une application (Gmail, Notion, Stripe…), transformer des données (extraire un champ, formater une date), appliquer une condition (si X alors Y sinon Z), ou appeler une IA (GPT-4o, Claude…). Vous chaînez autant de nodes que nécessaire.

Le Workflow — L'assemblage

Un workflow, c'est simplement l'enchaînement ordonné de nodes. Il peut être linéaire (A → B → C) ou ramifié avec des branches conditionnelles. Une fois activé, il tourne en autonomie complète — 24h/24, 7j/7.

4. Votre Premier Workflow en 10 Minutes

Passons à la pratique. Voici un workflow simple mais concret : chaque fois qu'un formulaire Typeform est soumis, ajouter automatiquement le contact dans Google Sheets et envoyer un email de confirmation. Idéal pour la gestion des leads entrants.

  • 1
    Créez un nouveau workflow vide

    Dans n8n, cliquez "New Workflow" puis "Add first step". Sélectionnez "Webhook" comme trigger (plus polyvalent que Typeform natif). Copiez l'URL générée.

  • 2
    Connectez votre source de données

    Dans Typeform (ou votre outil de formulaire), collez l'URL webhook dans les paramètres d'intégration. Soumettez un test pour voir les données arriver dans n8n en temps réel.

  • 3
    Ajoutez le node Google Sheets

    Cliquez "+" après le trigger, cherchez "Google Sheets", sélectionnez "Append Row". Connectez votre compte Google, sélectionnez votre spreadsheet. Mappez les champs (nom, email, message) avec les données du webhook.

  • 4
    Ajoutez le node Gmail pour l'email de confirmation

    Cliquez "+" après Google Sheets, cherchez "Gmail", sélectionnez "Send Email". Dans "To", mettez {{ $json.email }} (variable dynamique). Rédigez votre email de confirmation.

  • 5
    Activez et testez

    Cliquez le bouton "Active" en haut à droite. Soumettez un formulaire de test. Vérifiez que la ligne apparaît dans Sheets et que l'email arrive. Votre premier workflow est en production.

💡 Astuce d'exécution

Utilisez le mode "Test workflow" (bouton "Test" en bas) pendant la création pour voir les données circuler node par node en temps réel. Ça vous évite d'activer un workflow boggué en production.
